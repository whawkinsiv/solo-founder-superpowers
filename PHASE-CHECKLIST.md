# ICP-First SaaS Lifecycle Phase Checklist

Target ICP: solo, non-technical, bootstrapped founder building a real SaaS business.

## Phase 1: 0 -> $1k MRR

Goal: prove painful demand, ship fast, get first paying customers.

- [ ] `validate`
- [ ] `market-research`
- [ ] `customer-research`
- [ ] `prioritize`
- [ ] `pricing`
- [ ] `landing-page`
- [ ] `copywriting`
- [ ] `plan`
- [ ] `design`
- [ ] `navigation`
- [ ] `build`
- [ ] `database`
- [ ] `integrations`
- [ ] `secure`
- [ ] `test`
- [ ] `deploy`
- [ ] `payments`
- [ ] `analytics`
- [ ] `launch`
- [ ] `sales`
- [ ] `onboarding`
- [ ] `support`
- [ ] `monitor`
- [ ] `debug`
- [ ] `legal`
- [ ] `finances`
- [ ] `accounting`

Exit criteria:
- [ ] First repeatable channel for signups identified
- [ ] First paying customers acquired
- [ ] Onboarding and support loop functioning
- [ ] Baseline metrics dashboard in place (signup, activation, paid conversion)

## Phase 2: $1k -> $10k MRR

Goal: improve retention, tighten activation, and make acquisition repeatable.

- [ ] `retention`
- [ ] `feedback`
- [ ] `onboarding`
- [ ] `conversion`
- [ ] `email`
- [ ] `growth`
- [ ] `content`
- [ ] `social-media`
- [ ] `seo`
- [ ] `technical-seo`
- [ ] `ads` *(small-budget tests only)*
- [ ] `pricing` *(quarterly review)*
- [ ] `analytics`
- [ ] `monitor`
- [ ] `debug`
- [ ] `optimize`
- [ ] `support`
- [ ] `finances`

Exit criteria:
- [ ] Churn causes are identified and tracked
- [ ] At least one acquisition channel is predictably repeatable
- [ ] Trial/signup -> paid conversion improving month over month
- [ ] Cash runway and unit economics reviewed monthly

## Phase 3: $10k+ MRR

Goal: systemize operations, reduce founder bottlenecks, and scale safely.

- [ ] `hiring`
- [ ] `deploy` *(production hardening)*
- [ ] `payments` *(dunning, invoicing, leakage)*
- [ ] `secure`
- [ ] `accessibility`
- [ ] `support` *(self-serve scale)*
- [ ] `community`
- [ ] `prioritize`
- [ ] `pricing` *(packaging + expansion revenue)*
- [ ] `sales` *(for higher-ticket deals)*
- [ ] `ads` *(scale only if CAC payback works)*
- [ ] `ai-features`
- [ ] `analytics`
- [ ] `finances`
- [ ] `accounting`

Exit criteria:
- [ ] Core ops documented and delegable
- [ ] Billing failures and churn have active recovery systems
- [ ] Founder no longer required for every support/sales/build task
- [ ] Sustainable growth model validated (retention + acquisition + margin)
